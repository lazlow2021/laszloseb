Business Analysis

Key-points:

  • Agile business analysis
  • Based on case-study
  • Identify personas
  • Writing user-stories
  • Find the MVP

Business analysis always fun project for me. Find the obsticles and help the project flow is an essential task. In this short project I demonstrating how I would handle an Agile business analysis request.

Detail:

In this project I used a case study to make an agile business analysis. First I identify all personas. In this case study I only concentrated to the primary persona. I created a primary persona profile. After the profile was created I wrote the user stories for this persona. With the user story mapping technique I identified the requirements for this project. After I illustrated with an arrow the minimum viable product (MVP). I explained why these user stories are included the MVP. I choose two complex stories and illustrated how can be split into smaller stories. For the four user stories which been generated I wrote the acceptence criteria and with the MoSCoW prioritasion technique I prioritise the task.
